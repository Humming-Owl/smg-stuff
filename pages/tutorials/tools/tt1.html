Quick Intro

Before using some of the main SMG modding tools you need to install some dependencies on your system so that they work properly. The setup for the tools in this tutorial will be for all Whiteholes in existence that can be used in SMG1 modding.

(Fig. 0 - "Whitehole Despaghettification" with a galaxy opened)

Whitehole is the first and only (at least as of now) SMG level editor that exists and receives updates that is compatible with Windows/Linux/MacOS due to being Java written. It has a popular fame of not working properly mostly because of the setup for it to work wasn’t explained or clear to understand, also because there are several versions of Whitehole that people use and recommend (outdated ones) that lead to general confusion, and, essentially, because Java sucks. The relevant Whiteholes to mod SMG1 are Whitehole Despaghettification, Whitehole v1.7 and Whitehole for SMG1. The first 2 Whiteholes are the most used being Whitehole Despaghettification the one I recommend to use for SMG1/2 in general (known also as Whitehole Despag or only Despag), and, the last Whitehole (Whitehole for SMG1) is only used for a built-in Zone-Layer editor that is the only one that works (as of now) with SMG1.

NOTE: Keep in mind that the setup explanations can change overtime.


Java

Regarding the Java to be installed for each tool, you don’t need to install the JDK (Java Development Kit) as it is only the JRE (Java Runtime Environment) that is needed to run JAR files. To set the default Java on your system, there is a certain procedure depending on the platform you are using:


Set default Java on Windows

It is a mess to set the default JRE on Windows, what I do is to have all JREs offline installers I use, and, when I need to use one of them, I uninstall all of the others so that the default JRE is the only JRE left installed.

To check the default JRE being used type on a CMD window “java -version”.


Set default Java on Linux (Debian-based)

To set the default JRE on a Debian-based system type this command on terminal:

sudo update-alternatives --config java

and select the one you want to set as the default one. Check the “java -version” command output at the end.

Whitehole Despaghettification (By Aurum/SunakazeKun)

Recommended for general use with both SMG1/2.

Official Repository (GitHub) --> Link

Direct link to latest release (May 14, 2022 - “Despaghettification 140522” Pre-release) --> Link

This Whitehole version works with JRE 11 and greater.

- Windows users:

Get JRE 11 or greater from here (MSI installer) --> Link

- Linux users (Debian-based):

Get JRE 11 by typing this command on terminal:

sudo apt install openjdk-11-jre

This will install the JRE 11 that provides APT (at least on Ubuntu 18.04) (if you prefer, you can install instead the openjdk-17-jre package that provides the JRE 17 - it will also work).

- MacOS users:

Get JRE 11 or greater from here --> Link

- Object Database setup:

The object database file “objectdb.json” should be already in the “data” folder of the tool so you don’t have to set anything up for it.

Big Recommendation

If executing Despag or Whitehole v1.7 from Windows and you experience issues (galaxies not opening or part of the rendering screen of the program is blue) create a BAT file (In the same place as the "Whitehole.jar" file) (TXT file renamed to BAT) and paste this code on it:

java --add-exports=java.desktop/sun.awt=ALL-UNNAMED -Dsun.java2d.uiScale=1.0 -jar Whitehole.jar

Execute the BAT file and your issue may be corrected (double click on it).
